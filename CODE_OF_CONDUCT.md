# Code of Conduct

## Summary

Be technical. Be direct. Be sovereign. Disagree on merits, not people.

## Our standards

### Welcome

- Technical critique of code, architecture, and decisions
- Pointed questions about why something was built a certain way
- Firmly held disagreements that cite evidence (benchmarks, specs, prior art)
- Pushback on cloud lock-in, telemetry, or anything that erodes sovereignty
- Experienced contributors explaining tradeoffs directly and without
  unnecessary softening
- Newcomers asking questions, even if the answer exists somewhere in the docs

### Not welcome

- Personal attacks, insults, or sustained hostility directed at a person
  rather than an argument
- Harassment of any kind — on GitHub, Discord, email, or any other venue
- Discriminatory language or behavior targeting protected characteristics
- Publishing someone's private information without consent (doxxing)
- Sustained derailing of technical discussions into political / ideological
  tangents unrelated to the project
- Bad-faith rules-lawyering of this document to silence legitimate critique

## Scope

This applies to all project spaces — repository, issues, PRs, discussions,
chat, and any public representation of the project.

## Enforcement

Project maintainers will enforce this code. Actions, in escalating order:

1. Private clarification of expectations
2. Public correction in the affected thread
3. Temporary ban from participation (1–30 days)
4. Permanent ban for repeated or severe violations

Maintainers may skip steps for severe first-time violations (e.g. doxxing,
targeted harassment, malicious code submission).

## Reporting

Report violations privately to the project maintainers via GitHub (DM) or
email. Reports are kept confidential. Retaliation against reporters is itself
a violation.
